# Chapter 4 Pretrained Conformer Bundle

This directory is intentionally small and explicit. It exists so CPU reproduction
lanes can rebuild data/protocol artifacts and then run Conformer inference or
figure generation without retraining Conformer models on CPU.

Default policy:

- `device=cpu` uses these pretrained artifacts unless
  `ECG_ALLOW_CPU_CONFORMER_TRAIN=1` is set.
- `device=cuda` keeps the existing training path.
- Raw data, splits, SQI features, source-data tables, and figures are still
  generated by the pipeline.

Contents:

- `seta_e31_leadwise_shared/`: Set-A E31 checkpoint plus training history. CPU
  reproduction reloads `best_model.pt` and regenerates predictions for the
  freshly built Set-A protocol.
- `but_e31_query_mean_fused_conformer/`: BUT v116 query-mean fused E31
  checkpoint and predictions used by Chapter 4 model comparison and patching
  analyses.
- `but_architecture_ablation/`: compact seed/candidate checkpoints,
  predictions, and train logs for Fig. M7-style architecture ablation. Large
  split atlases are not bundled; `split_audits.json` records the validated
  train/test counts used by the audit.

`manifest.json` records byte sizes and SHA-256 checksums.
